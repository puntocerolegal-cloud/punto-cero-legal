═══════════════════════════════════════════════════════════════════════════════
OPCIÓN 1: EJECUTAR EN LINUX/MAC (BASH)
═══════════════════════════════════════════════════════════════════════════════

Abre terminal en la raíz del proyecto y ejecuta:

  chmod +x deploy-automation.sh
  ./deploy-automation.sh

Qué hace:
  ✅ Detecta/inicializa Git
  ✅ Realiza commit automático de cambios
  ✅ Pushea a GitHub main
  ✅ Compila frontend (npm run build)
  ✅ Verifica configuración Vercel
  ✅ Proporciona instrucciones para Vercel Dashboard

Tiempo: ~5-10 minutos

═══════════════════════════════════════════════════════════════════════════════
OPCIÓN 2: EJECUTAR EN WINDOWS (POWERSHELL)
═══════════════════════════════════════════════════════════════════════════════

Abre PowerShell como Administrador y ejecuta:

  Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process
  .\deploy-automation.ps1

O simplemente:

  powershell -ExecutionPolicy Bypass -File .\deploy-automation.ps1

Qué hace:
  (Idéntico a Bash - versión Windows)

Tiempo: ~5-10 minutos
