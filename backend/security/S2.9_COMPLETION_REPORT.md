## 🧠 The Five Core Engines

### 1. **Security Governor Engine**
**File:** `security_governor_engine.py` (315 lines)

**Purpose:** Validate and control S2.8 autonomous actions.

**Global Safety Limits:**
```
max_users_isolated_per_minute: 10
max_tenants_in_lockdown: 3
max_blocks_per_second: 20
max_rate_limits_per_minute: 100
max_ips_blocked_per_minute: 5
max_autonomous_actions_per_minute: 50
```

**Decision Logic:**
- Receives action from S2.8
- Checks against global limits
- Returns: APPROVED, DOWNGRADED, REJECTED, or DEFERRED
- CRITICAL risk (>95) can override limits
- Otherwise: downgrade action if limits exceeded

**Example Flow:**
```python
governor.validate_action(
    action_type="isolate_user",
    target="user_123",
    risk_score=78,
)
# Returns:
# {
#   approval_status: "downgraded",
#   final_action: "block_user",  # Downgraded from isolate
#   reason: "Exceeded max_users_isolated_per_minute",
# }
```

### 2. **Policy Arbitration Engine**
**File:** `policy_arbitration_engine.py` (279 lines)

**Purpose:** Resolve conflicts between security layers.

**Priority Hierarchy:**
```
1. S2.9 Governor      (HIGHEST)
2. S2.5 GSCL
3. S2.8 Autonomous
4. S2.6 Intelligence  (LOWEST)
```

**Conflict Resolution:**
When layers disagree on what to do, higher-priority layer wins.

```python
arbitration = arbitrate(
    gscl_decision="ALLOW",      # GSCL says allow
    s2_6_decision="MONITOR",    # Intelligence suggests monitor
    s2_8_decision="BLOCK",      # Autonomous wants to block
    s2_9_decision="THROTTLE",   # Governor says throttle
)
# Final decision: THROTTLE (governor priority highest)
```

**Key Features:**
- Tracks all decision sources
- Generates reasoning trace
- Detects conflicts between layers
- Statistics on layer agreement/disagreement

### 3. **Circuit Breaker Manager**
**File:** `circuit_breaker_manager.py` (308 lines)

**Purpose:** Prevent system self-destruction through cascading failures.

**States:**
```
NORMAL
  ↓ (if warning triggers)
WARNING
  ↓ (if critical triggers)
ACTIVE_BREAK
  ↓ (if multiple critical issues)
EMERGENCY_LOCKDOWN
```

**Triggers:**
- Excessive blocks (> 5% of users)
- Tenant mass isolation (> 3 tenants locked)
- False positive spike (> 15% FP rate)
- Anomaly engine instability (< 0.8 stability)
- SOC overload (> 10k event backlog)

**Automatic Actions:**
```
ACTIVE_BREAK:
  • Disable S2.8 autonomous BLOCKING
  • Switch to MONITOR_ONLY mode
  • Escalate to SOC

EMERGENCY_LOCKDOWN:
  • Trigger fail-safe mode (global)
  • Block all writes
  • Allow reads only
  • Send CRITICAL alert
```

**Auto-Recovery:**
- Monitors health continuously
- Auto-resets after configured duration (10-30 min)
- Manual override capability always available

---

### 4. **System Risk Governor**
**File:** `system_risk_governor.py` (334 lines)

**Purpose:** Monitor global system health and adjust autonomy level.

**Health Metrics:**
- Global system risk score (0-100)
- False positive rate
- False negative rate
- Autonomy safety score
- Enforcement effectiveness
- Response accuracy

**Autonomy Level Adjustment:**
```
Safety > 85   → Level 1.0 (FULL AUTONOMOUS)
Safety 70-85  → Level 0.8 (HIGH AUTONOMOUS)
Safety 50-70  → Level 0.5 (MIXED: SOC review required)
Safety < 50   → Level 0.2 (LOW: Mostly manual)
```

**Tenant Risk Distribution:**
- Tracks risk per tenant
- Detects high-risk tenants
- Enables per-tenant policy adjustment

**Example:**
```
System health:
  Global risk: 35/100
  FP rate: 8% (low)
  FN rate: 2% (good)
  Enforcement effective: 94%
  → Autonomy level = 0.8 (HIGH AUTONOMOUS)
```

## 🔗 How S2.9 Controls S2.8

### Request Flow with S2.9

```
Request
  ↓
S2.5 GSCL (enforcement)
  ↓
S2.6 Intelligence (analysis) → generates risk_context
  ↓
S2.8 Autonomous Response (proposes action)
  │
  ├─→ decision = decide(risk_context)
  └─→ proposed_action = BLOCK/ISOLATE/etc
  ↓
🚨 S2.9 GOVERNOR VALIDATION ← NEW
  │
  ├─→ governor.validate_action(proposed_action)
  │
  ├─→ Is within global limits?
  │     YES → APPROVED
  │     NO  → DOWNGRADED or REJECTED
  │
  └─→ arbitration.arbitrate(
      gscl=..., s2_6=..., s2_8=..., s2_9=...
    )
  ↓
FINAL_DECISION (S2.9 can override S2.8)
  ↓
S2.7 SOC (observe + log)
  ↓
Response (allow/throttle/block/isolate)
```
