## 🧠 The Five Core Engines

### 1. **Autonomous Decision Engine**
Maps risk signals (0-100) to 6 decision types with confidence scoring.

```
Risk 0-39   → ALLOW (continue)
Risk 40-59  → MONITOR (increased logging)
Risk 60-74  → THROTTLE (rate limit)
Risk 75-84  → BLOCK (deny access)
Risk 85-94  → ISOLATE_USER (sandbox)
Risk 95-100 → ISOLATE_TENANT + FAIL_SAFE (full isolation)
```

**Key Feature:** Confidence scoring ensures only high-confidence decisions auto-execute.

---

### 2. **Mitigation Engine**
Executes real-time security actions: blocking, throttling, revocation, isolation.

**Supported Actions:**
- Rate limit users
- Block users/IPs
- Quarantine tenants
- Revoke tokens
- Reduce permissions
- Isolate users to sandbox
- Trigger fail-safe mode
- Notify SOC

**Key Feature:** All actions logged to SOC immediately, with expiry management.

---

### 3. **Auto-Policy Engine**
Dynamically adjusts security policies at runtime without redeployment.

**Policy Modes:**
- **NORMAL** — Standard operations (1000 req/min, 100 connections)
- **ELEVATED** — No deletes, MFA required (500 req/min)
- **STRICT** — Read-only, IP whitelist (100 req/min)
- **LOCKDOWN** — Minimal access, audit all (10 req/min)

**Key Feature:** Can escalate entire tenant's security posture in milliseconds.

---

### 4. **Containment Engine**
Isolates threats without affecting the entire system.

**Containment Scopes:**
- **User** → Sandbox individual user
- **Tenant** → Quarantine entire tenant
- **Endpoint** → Disable specific endpoint
- **Session** → Sandbox suspicious session

**Containment Modes:**
- **MONITORED** — Log all activity
- **SANDBOX** — Return fake DB responses
- **READONLY** — Block all writes
- **DISABLED** — Return 503 error

**Key Feature:** Malicious user sees locked-down interface while system continues normally.

---

### 5. **Recovery Engine**
Automatically restores system to normal after threats are contained.

**Recovery States:**
```
ACTIVE_THREAT
    ↓
THREAT_CONTAINED
    ↓
SAFE_RECOVERY
    ↓
GRADUAL_RESTORE (permissions restored in stages)
    ↓
FULL_RESTORE (complete recovery)
```

**Key Feature:** Phased restoration prevents another attack during recovery.

## 🔄 How It Works

### The Autonomous Loop

```
1. DETECT (S2.6 Intelligence)
   ↓ Risk signals from behavior + attack graphs
   
2. DECIDE (S2.8 Decision Engine)
   ↓ Maps risk → decision with confidence scoring
   
3. ACT (S2.8 Mitigation + Containment)
   ↓ Executes actions immediately
   
4. OBSERVE (S2.7 SOC)
   ↓ Logs all actions in real-time
   
5. LEARN (S2.6 Feedback)
   ↓ Improves models for next threat
```

**Result:** Threats are detected, analyzed, mitigated, and logged within milliseconds.

## ✨ Key Innovations

1. **Confidence Scoring** — Not all decisions are equal. High-confidence decisions auto-execute, low-confidence ones require SOC review.

2. **Gradual Mitigation** — Instead of binary allow/deny, system can throttle, monitor, or partially isolate while maintaining business continuity.

3. **Policy Adaptation** — Security policies change dynamically at runtime without redeployment. Can go from NORMAL → LOCKDOWN in milliseconds.

4. **Threat Containment** — Malicious users see a sandboxed, fake interface while actual system continues normally. They believe they're accessing the system but can't.

5. **Autonomous Recovery** — After threat is contained, system automatically restores in stages, preventing another attack during recovery.

6. **Zero Data Loss** — No destructive actions. Everything is reversible. Data preserved for forensics.
