## TAREA 1: CRONJOB MEJORADO {#tarea-1}

### Problema Identificado
```
❌ Cronjob ejecutaba en proceso principal de FastAPI
   - Riesgo de crear múltiples instancias
   - Posibilidad de tareas duplicadas
   - Podría bloquear requests
```

### Solución Implementada

**Ubicación:** `backend/services/cron_jobs.py`

#### Antes
```python
_scheduler: Optional[CronScheduler] = None

async def init_cron_scheduler(db):
    global _scheduler
    _scheduler = CronScheduler(db)
    await _scheduler.start()
```

#### Después
```python
_scheduler: Optional[CronScheduler] = None
_scheduler_lock = asyncio.Lock()  # Singleton lock

async def init_cron_scheduler(db):
    global _scheduler
    async with _scheduler_lock:  # Garantiza una sola instancia
        if _scheduler and _scheduler.is_running:
            logger.info("Cron scheduler already running, skipping")
            return
        
        if _scheduler is None:
            _scheduler = CronScheduler(db)
        
        await _scheduler.start()
        logger.info("Cron scheduler initialized (singleton)")
```

### Validaciones Implementadas

✅ **Una sola instancia**
- Mutex (`asyncio.Lock`) protege inicialización
- Check `if _scheduler.is_running` previene duplicados
- Reintentos seguros

✅ **Sin tareas duplicadas**
- Singleton lock en init
- Check `is_running` en shutdown
- Logging de cada operación

✅ **Sobrevive reinicios**
- Scheduler registrado en `@app.on_event("startup")`
- Shutdown limpio en `@app.on_event("shutdown")`
- Estado persistido en MongoDB

✅ **Registra logs y errores**
- Logs en info/warning/error
- Auditoría en `audit_logs`
- Error tracking en `error_tracker`

#### Archivos Modificados
```
backend/services/cron_jobs.py (líneas 240-280)
  - Agregado _scheduler_lock
  - Mejorado init_cron_scheduler()
  - Mejorado shutdown_cron_scheduler()
```
