# PR-06.1 — Especificación Técnica: Protección localStorage con try/catch

## Evidencia del Problema

**Código actual SIN try/catch — Acceso directo a localStorage:**

```js
// ❌ Sin protección
const raw = localStorage.getItem(STORAGE_KEY);
if (raw) return { ...DEFAULT, ...JSON.parse(raw) };
// Si localStorage.getItem() lanza SecurityError, código se detiene
// Si JSON.parse() lanza SyntaxError, código se detiene
```

**Código existente CON try/catch — Patrón a seguir:**

```js
// ✓ Con protección (ya existe en el proyecto)
try {
  const payload = STORAGE_PASSPHRASE ? await encryptString(token) : token;
  localStorage.setItem(TOKEN_KEY, payload);
} catch (e) {
  console.error('Failed to store token securely:', e);
  localStorage.setItem(TOKEN_KEY, token);  // Fallback
}
```

**Inconsistencia:** El proyecto TIENE el patrón de manejo en AuthContext, pero NO lo aplica en 6 archivos más.

---

## Excepciones Posibles por Navegador

Navegadores pueden lanzar las siguientes excepciones al acceder localStorage:

| Excepción | Causa | Contexto |
|-----------|-------|---------|
| `SecurityError` | Acceso denegado (Private Mode, política CORS, dominio no permitido) | Firefox Private, Safari Private, restricciones corporativas |
| `QuotaExceededError` | localStorage lleno (límite típico 5-10MB) | Datos acumulados, base de datos corruptos |
| `SyntaxError` | JSON.parse() con string inválido (datos corruptos) | localStorage con data malformada |
| `TypeError` | Acceso a null o undefined | No aplica (localStorage está disponible pero puede devolver null) |


**Excepciones posibles:**
- `localStorage.getItem()` → `SecurityError`, `QuotaExceededError`

**Fallback esperado:** `'0'` (fallback del operador `||`), por lo tanto `parseInt('0') = 0`, por lo tanto `Date.now() < 0 = false`

**Cambio necesario:**
```js
const [dismissed, setDismissed] = useState(() => {
  try {
    const until = parseInt(localStorage.getItem(UPGRADE_DISMISS_KEY) || '0', 10);
    return Date.now() < until;
  } catch (e) {
    return false;  // Por defecto, mostrar banner (no descartado)
  }
});
```
