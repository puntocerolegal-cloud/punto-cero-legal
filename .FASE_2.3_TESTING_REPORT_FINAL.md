## ERRORES ENCONTRADOS {#errores-encontrados}

### Críticos
```
❌ NINGUNO ENCONTRADO
```

### Altos
```
⚠️ 1. Cronjob ejecuta en proceso principal de FastAPI
   - En producción, considerar APScheduler o Celery separado
   - Impacto: Puede bloquear requests si toma mucho tiempo
   - Severidad: ALTO
   - Recomendación: Implementar job queue separado

⚠️ 2. Sin validación explícita de APP_PUBLIC_URL
   - Si es localhost, MP no puede enviar webhooks
   - Impacto: Webhooks no llegarán en development
   - Severidad: ALTO
   - Recomendación: Validar en startup
```

### Medios
```
⚠️ 3. webhook_handler importado dinámicamente en endpoint
   - `from services.webhook_handler import ...` dentro de la función
   - Impacto: Pequeño overhead per-request
   - Severidad: MEDIO
   - Recomendación: Importar en top del archivo

⚠️ 4. Sin timeout explícito en algunas operaciones MongoDB
   - El cliente motor tiene default, pero no visible en código
   - Impacto: Queries podem quedar colgadas
   - Severidad: MEDIO
   - Recomendación: Agregar timeout explícito

⚠️ 5. Logs sin rotación configurada
   - webhook_logs + webhook_events crecen indefinidamente
   - Impacto: Storage puede agotarse
   - Severidad: MEDIO
   - Recomendación: Implementar TTL en colecciones (creado en FASE 2.2: cleanup de 30 días)
```

### Bajos
```
ℹ️ 6. No hay circuit breaker para MP API
   - Si MP está caído, requests pueden fallar
   - Impacto: Experiencia de usuario degradada
   - Severidad: BAJO
   - Recomendación: Opcional para fase beta

ℹ️ 7. Email notifications no enviadas aún
   - Solo guardan en db.notifications
   - Impacto: Admin/usuarios no reciben emails
   - Severidad: BAJO
   - Recomendación: Implementar provider (SendGrid, AWS SES)
```
