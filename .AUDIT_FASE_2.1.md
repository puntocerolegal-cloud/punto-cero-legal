## 8. AUTENTICACIÓN Y SEGURIDAD

### Validaciones Webhook

```python
# ✅ Valida evento de pago
if topic != "payment":
    return {"received": True, "ignored": True}

# ✅ Valida ID de pago
if not mp_payment_id:
    return

# ✅ Valida token de MP
if not token:
    return {"received": True, "no_token": True}

# ✅ Valida respuesta de MP
if r.status_code != 200:
    return {"received": True, "lookup_failed": r.status_code}
```

### NO VALIDA

```
❌ Firma HMAC de MP (cualquiera puede enviar)
❌ IP origen de MP
❌ Timestamp del evento
❌ Duplicados (detecta por status == "paid" solamente)
```

## 13. PROBLEMAS IDENTIFICADOS

### CRÍTICOS

1. **❌ Dos URLs de webhook**
   - MP enviará a una URL configurable
   - Hay dos endpoints: `/api/payment/webhook` y `/api/invoices/webhook/mercadopago`
   - SOLUCIÓN: Consolidar en una sola URL

2. **❌ Sin validación de firma HMAC**
   - Cualquiera puede enviar webhooks falsos
   - SOLUCIÓN: Implementar validación de firma

3. **❌ Sin idempotencia en invoices webhook**
   - Puede procesar mismo webhook dos veces
   - SOLUCIÓN: Guardar event_id recibido, rechazar duplicados

4. **❌ Sin historial de webhooks**
   - No hay `webhook_logs` collection
   - SOLUCIÓN: Guardar cada webhook recibido para debugging

### ALTOS

5. **❌ Eventos incompletos**
   - Solo procesa "payment.approved"
   - NO procesa: subscription.*, refund.*, chargeback.*
   - SOLUCIÓN: Implementar todos los eventos

6. **❌ Sin cronjob de renovación**
   - `renewal_service.py` existe pero no se ejecuta automáticamente
   - SOLUCIÓN: Registrar en Docker/SystemD

7. **❌ Sincronización incompleta**
   - `subscription_expires_at` nunca se calcula
   - `renewal_status` no existe
   - SOLUCIÓN: Actualizar campos en `db.users`

### MEDIOS

8. **⚠️ APP_PUBLIC_URL es crítica**
   - Si es localhost, MP no puede enviar webhooks
   - SOLUCIÓN: Validar en startup

9. **⚠️ Sin validación de transiciones de estado**
   - Puede saltar estados (trial → cancelled sin pasar por active)
   - SOLUCIÓN: Implementar máquina de estados

10. **⚠️ Sin reintentos en Mongo**
    - Si falla insert, webhook se pierde
    - SOLUCIÓN: Implementar reintentos con backoff
