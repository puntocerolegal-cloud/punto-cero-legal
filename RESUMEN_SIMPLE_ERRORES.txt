ERROR 4: Sin Fallback (Si MongoDB Falla = App Muere)
────────────────────────────────────────────────────

¿QUÉ PASABA?
  Si MongoDB no conecta → db = <objeto de conexión fallida>
  Luego: await db.users.find_one(...) → ERROR (no hay conexión)
  App crashea sin graceful fallback

LÍNEA DEL PROBLEMA:
  Línea 227: existing = await db.users.find_one(...)
             ↓ Si db=None o connection failed: CRASH

SOLUCIÓN:
  Línea 220+: Ahora cada hook verifica primero:
             
             if db is None:
                 logger.warning("Skipping X: DB not available")
                 return  # ← Salir gracefully
             
             # ... resto del código

RESULTADO:
  ✅ Si DB no está disponible → hooks se saltan
  ✅ App sigue viva (no crashea)
  ✅ Endpoints retornan 503 (en lugar de timeout)
  ✅ Cuando MongoDB vuelve → app recupera automáticamente
