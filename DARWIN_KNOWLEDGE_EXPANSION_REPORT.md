## PHILOSOPHY & ARCHITECTURE

### The Core Principle
**"Knowledge First, AI Second"**

This means:
- Punto Cero defines what to say
- Darwin/AI expresses it naturally
- AI never improvises
- Knowledge is the source of truth
- AI amplifies, not replaces

### Why This Matters
- **Consistency:** All responses grounded in same truth
- **Quality:** Crafted knowledge > AI hallucinations
- **Control:** We decide message, not algorithm
- **Scalability:** New verticals reuse same knowledge
- **Transparency:** Clear source for every statement

### The Architecture
```
KNOWLEDGE LIBRARY (Punto Cero truth)
         ↓
Darwin Reads & Understands
         ↓
Darwin Expresses Naturally
         ↓
Customer Receives Knowledge + Natural Language
```

## DOCUMENT STRUCTURE PATTERN

Each document follows this pattern:

```
1. OBJECTIVE
   Why this document exists

2. FRAMEWORK/STRUCTURE
   How to think about this topic

3. CATEGORIES/ELEMENTS
   Specific breakdowns

4. USAGE EXAMPLES
   How Darwin uses this

5. METADATA
   Owner, frequency, dependencies

6. PENDING CONTENT
   What still needs writing
```

This pattern makes documents:
- Easy to understand
- Easy to expand
- Easy for AI to parse
- Easy to maintain
- Future-proof

## HOW DARWIN USES THIS KNOWLEDGE

### Example 1: Customer Asks "Why should I use you?"

**Old approach (without knowledge):**
Darwin improvises answer based on training

**New approach (with knowledge):**
1. Darwin checks value_proposition.md
2. Finds relevant framework: Problem → Transformation → Benefit
3. Consults products_transformation.md for specific product
4. Checks communication_style.md for tone
5. Expresses answer naturally using exact knowledge

**Result:** Consistent, accurate, on-brand response every time

### Example 2: Customer Is Frustrated

**Old approach (without knowledge):**
Darwin guesses how to respond emotionally

**New approach (with knowledge):**
1. Darwin checks human_empathy.md
2. Finds "FRUSTRATED CUSTOMER" section
3. Learns principle: "Validate frustration" + "Show understanding" + "Provide proof"
4. Crafts response using that framework
5. Checks communication_style.md for language

**Result:** Emotionally intelligent, consistent response

## GOVERNANCE

### Ownership Structure

**Foundational Docs (Company Story, Principles)**
- Owner: Leadership
- Update: Yearly
- Approval: Board

**Strategic Docs (Vision)**
- Owner: Strategy Team
- Update: Quarterly
- Approval: Leadership

**Operational Docs (All others)**
- Owner: Respective teams
- Update: Monthly-Quarterly
- Approval: Department head

### Update Process

1. **Owner identifies** change needed
2. **Document updated** in living system
3. **Related docs checked** for consistency
4. **Team notified** of change
5. **Darwin re-indexed** automatically
