## RIESGOS POTENCIALES

### Críticos
```
❌ Si HMAC no valida → webhooks falsos aceptados
❌ Si event_id no es UNIQUE → datos duplicados
❌ Si cron no ejecuta → renovaciones no automáticas
```

**Mitigación:** Todos validados en código

### Altos
```
⚠️ Si Mercado Pago webhook URL mal configurada → eventos perdidos
⚠️ Si MongoDB indexing falla → queries lentos
⚠️ Si sincroni zación incompleta → datos inconsistentes
```

**Mitigación:** Validable en testing

### Medios
```
⚠️ Si rate limiting muy estricto → falsos positivos
⚠️ Si logs en disco sin rotación → storage agotado
⚠️ Si cron jobs se solapan → duplicación de trabajo
```

**Mitigación:** Configurables en deployment
