## FASE 5: WEBHOOKS Y NOTIFICACIONES

### Webhook de Mercado Pago

**URL:** `POST /api/payment/webhook`

**Parámetros MP:**
- `type=payment` o `topic=payment`
- `data.id=XXXX` (MP payment ID)

**Lógica:**
1. ✅ Valida que sea un evento de pago
2. ✅ Consulta estado en MP API: `GET /v1/payments/{mp_payment_id}`
3. ✅ Si status = "approved" → `_apply_payment_success()`
4. ✅ Si status = "rejected" o "cancelled" → notifica admin
5. ✅ Devuelve siempre 200 OK (acuso de recibo)
6. ✅ Maneja errores gracefully

## FASE 10: ESTADOS DE SUSCRIPCIÓN

### Estados Válidos en `users.subscription_status`

```
trial              → Período de prueba (7 días)
active             → Pagó, suscripción activa
pending_payment    → En proceso de pago (checkout abierto)
pending_verification → Pago manual, esperando verificación admin
suspended          → Suspendida (deuda, problema de pago)
cancelled          → Cancelada por el usuario
expired            → Vencida (no renovó a tiempo)
```

**Transiciones Válidas:**
- `trial` → `active` (pago exitoso)
- `trial` → `cancelled` (canceló sin pagar)
- `active` → `suspended` (deuda)
- `active` → `cancelled` (canceló)
- `suspended` → `active` (pagó deuda)
- `cancelled` → `active` (reactivó)
- `expired` → `active` (renovó)
