## FASE 2: VALIDACIÓN HMAC

### Implementación Oficial Mercado Pago

```python
async def validate_hmac_signature(payload: str, signature: str) -> bool:
    """
    Valida HMAC-SHA256 de Mercado Pago.
    
    Formato MP: X-Signature: ts=1234567890,v1=hexdigest
    Payload: id={event_id}&type={event_type}
    Secret: MP_ACCESS_TOKEN
    """
```

**Flujo:**
1. ✅ Extraer payload: `id={event_id}&type={event_type}`
2. ✅ Extraer firma de header `X-Signature`
3. ✅ Calcular HMAC esperado: `HMAC-SHA256(payload, secret)`
4. ✅ Comparación timing-safe (prevenir timing attacks)
5. ✅ Si inválida: rechazar con 401 + log

**Resultado:**
- ❌ Webhooks con firma inválida: **RECHAZADOS**
- ✅ Intento registrado en webhook_logs
- ✅ Admin notificado de intentos fallidos

## FASE 3: IDEMPOTENCIA

### Nueva Colección: `webhook_events`

```javascript
{
  event_id: "12345",                   // Único por evento
  type: "payment.approved",
  action: "success",                   // success|error|duplicate|invalid_signature
  processed: true,
  processed_at: ISODate(...),
  retries: 0,
  payload_hash: "abc123...",           // SHA256 del payload
  error: null,
  created_at: ISODate(...)
}
```

**Índices:**
- `event_id` (UNIQUE) — búsqueda O(1)
- `type`, `processed`, `created_at`

**Flujo anti-duplicados:**
1. ✅ Recibir webhook
2. ✅ Verificar: ¿event_id ya existe?
3. ✅ Sí → responder "duplicate", sin ejecutar lógica
4. ✅ No → procesar evento

**Beneficio:** Si MP envía el mismo webhook 10 veces → se procesa **UNA VEZ**.

## FASE 7: CRON JOB REGISTRADO

### Nueva Clase: `CronScheduler`

**Ubicación:** `backend/services/cron_jobs.py`

**Tareas Automáticas:**

#### Diariamente (00:00 UTC)
```
✅ check_and_renew_subscriptions()     → Detectar próximas a vencer
✅ retry_failed_renewals()              → Reintentar fallidas
✅ _mark_expired_subscriptions()        → Marcar como expiradas
```

#### Cada 6 horas
```
✅ _notify_near_expiration()            → Notificar próximas a vencer
```

#### Semanalmente (lunes 02:00 UTC)
```
✅ _cleanup_old_webhook_logs()          → Eliminar logs > 30 días
✅ _cleanup_old_webhook_events()        → Eliminar eventos procesados > 30 días
✅ _generate_health_report()            → Generar reporte
```

**Integración en FastAPI:**
```python
@app.on_event("startup")
async def init_cron_jobs():
    await init_cron_scheduler(db)

@app.on_event("shutdown")
async def shutdown_cron_jobs():
    await shutdown_cron_scheduler()
```

**Estado:** ✅ Registrado automáticamente en startup

### Estrategia de Recuperación

```

  ✅ Registrar en webhook_logs
  ✅ Registrar en audit_logs
  ✅ Responder 200 OK (acuso de recibo)
  ✅ Marcar evento como procesado con error
  ✅ Admin debe revisar webhook_logs

Si timeout Mercado Pago:
  ✅ Fallar gracefully
  ✅ Responder 200 OK
  ✅ Loguear error
  ✅ Manualreviewer puede reintentar

Nunca:
  ❌ Perder el evento
  ❌ Dejar estados intermedios
  ❌ Duplicar procesamiento
```

## FLUJO COMPLETO WEBHOOK

```
1. Mercado Pago envía evento
   ↓
2. POST /api/payment/webhook
   ↓
3. Validar HMAC (si inválido → rechazar 401)
   ↓
4. Verificar si event_id ya procesado (si sí → retornar 200 "duplicate")
   ↓
5. Procesar evento (payment, subscription, refund, etc)
   ↓
6. Actualizar MongoDB (transacciones, usuarios, auditoría)
   ↓
7. Registrar en webhook_events (marca como procesado)
   ↓
8. Registrar en webhook_logs (para auditoría)
   ↓
9. Responder 200 OK con metrics
   ↓
10. Notificar admin si hay problemas
```
