## RESUMEN DE CAMBIOS

### Backend — `backend/routes/payment.py`

#### Nuevos Endpoints Implementados

1. **GET `/api/payment/subscription-status`**
   - Devuelve estado actual de la suscripción del usuario autenticado
   - Incluye: plan activo, estado, próxima renovación, acciones disponibles
   - Respuesta: `SubscriptionStatusResponse`

2. **POST `/api/payment/renew`**
   - Renueva una suscripción vencida o próxima a vencer
   - Crea nuevo payment intent en Mercado Pago
   - Soporta último ciclo (monthly/annual) usado
   - Tipo de transacción: `renewal`

3. **POST `/api/payment/change-plan`**
   - Cambia de plan con prorrateo automático
   - Si plan superior: cobra diferencia
   - Si plan inferior: crea crédito para próxima renovación
   - Si mismo precio: cambio inmediato
   - Tipo de transacción: `plan_change`

4. **POST `/api/payment/cancel`**
   - Cancela suscripción activa
   - Marca como `cancelled` con fecha de cancelación
   - Registra en auditoría
   - Notifica admin

5. **POST `/api/payment/reactivate`**
   - Reactiva una suscripción cancelada
   - Crea nuevo payment intent
   - Permite restaurar acceso al plan anterior
   - Tipo de transacción: `reactivation`

### Backend — `backend/services/renewal_service.py` (NUEVO)

Servicio de renovación automática de suscripciones:

#### Funciones Principales

1. **`check_and_renew_subscriptions(db)`**
   - Ejecuta diariamente (cronjob)
   - Busca usuarios con suscripción próxima a vencer (próximos 7 días)
   - Crea payment intents automáticamente en MP
   - Notifica usuarios
   - Retorna stats: {checked, needs_renewal, auto_renewed, failed}

2. **`retry_failed_renewals(db)`**
   - Reintenta renovaciones que fallaron
   - Máximo 3 reintentos por transacción
   - Si lleva > 3 días pendiente, crea nuevo intent
   - Para renovaciones automáticas solamente

#### Features

- ✅ Detección automática de suscripciones a vencer
- ✅ Creación de preferencias reales en Mercado Pago
- ✅ Notificaciones a usuarios
- ✅ Reintentos automáticos
- ✅ Logging y auditoría
- ✅ Manejo de errores robusto

### Backend — `backend/routes/billing_admin.py` (NUEVO)

Panel administrativo para gestión de suscripciones:

#### Endpoints

1. **GET `/api/billing-admin/subscriptions`**
   - Listar todas las suscripciones activas
   - Filtros: status, plan_id, country
   - Retorna: email, nombre, plan, estado, país

2. **GET `/api/billing-admin/transactions`**
   - Listar transacciones de pago
   - Filtros: status, type
   - Tipos: payment, renewal, plan_change, reactivation

3. **GET `/api/billing-admin/dashboard`**
   - Dashboard de facturación completo
   - KPIs: activos por estado, por plan, MRR, ARR, churn
   - Ingresos últimos 30 días
   - Pagos pendientes

4. **POST `/api/billing-admin/trigger-renewal-check`**
   - Ejecuta manualmente la verificación de renovaciones
   - Para testing/debugging
   - Retorna stats de ejecución

5. **POST `/api/billing-admin/trigger-retry-failed`**
   - Ejecuta manualmente reintentos fallidos
   - Para testing/debugging

6. **GET `/api/billing-admin/pending-renewals`**
   - Listar renovaciones automáticas pendientes
   - Útil para debugging
   - Incluye checkout URLs

7. **POST `/api/billing-admin/verify-receipt/{receipt_id}`**
   - Aprobar/rechazar comprobantes de pago manual
   - action: approve | reject
   - Al aprobar: activa suscripción del usuario
   - Notifica al usuario

## FLUJOS IMPLEMENTADOS

### 1. Renovación Automática
```
Cronjob diario
    ↓
check_and_renew_subscriptions()
    ↓
Buscar usuarios: próxima renovación ≤ 7 días
    ↓
Para cada usuario:
    - Crear preference en Mercado Pago
    - Guardar transaction (type=auto_renewal, status=pending)
    - Notificar usuario
    ↓
Retornar stats
```

### 2. Renovación Manual
```
Usuario hace POST /api/payment/renew
    ↓
Validar: subscription_status = "expired" o "trial"
    ↓
Obtener último billing_cycle usado
    ↓
Crear preference en MP
    ↓
Guardar transaction (type=renewal, status=pending)
    ↓
Retornar checkout_url
```

### 3. Cambio de Plan
```
Usuario hace POST /api/payment/change-plan
    ↓
Validar: plan diferente, subscription_status = "active"
    ↓
Calcular prorrateo: (precio_nuevo - precio_viejo) × (días_restantes/30)
    ↓
Si prorrateo < 0: crear crédito para próxima renovación
Si prorrateo > 0: crear preference en MP (tipo=plan_change)
Si prorrateo = 0: cambio inmediato
    ↓
Retornar acción (credit, payment_required, instant)
```

### 4. Cancelación
```
Usuario hace POST /api/payment/cancel
    ↓
Validar: subscription_status = "active"
    ↓
Marcar como "cancelled"
    ↓
Guardar cancelled_at
    ↓
Registrar en audit_logs
    ↓
Notificar admin
```

### 5. Reactivación
```
Usuario hace POST /api/payment/reactivate
    ↓
Validar: subscription_status = "cancelled"
    ↓
Crear preference en MP (tipo=reactivation)
    ↓
Guardar transaction
    ↓
Notificar usuario
```
