🔄 CONVERSATION FLOW ARCHITECTURE
═════════════════════════════════════════════════════════════════════════════

Message Flow:
  User Input → Channel Adapter → Router → Agent → Response Builder → Logger → Output

Processing:
  1. Channel receives message
  2. Router analyzes and routes
  3. Memory loads context
  4. Agent processes
  5. Response formatted
  6. Logger records
  7. Response sent

Data Flow:
  Message → ChannelMessage → RoutingDecision → AgentResponse → FormattedResponse
